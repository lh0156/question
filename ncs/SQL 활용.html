# USER_CONSTRAINTS
- 생성된 테이블을 사용하여 레코드를 추가 할 때 개발자가 원하는 형태의 데이터를 받기 위해 제약사항을 설정된 제약사항을 확인하는 Dictionary


# USER_OBJECTS
- 접속한 계정이 생성한 모든 DBMS객체를 확인 할 때 사용하는 Dictionary


# truncate
- 테이블내의 모든 레코드를 절삭한다. delete 명령어와의 차이점은 delete 명령어는 레코드를 삭제 후 트랜잭션에 의해서 삭제된 레코드를 복구할 수 있지만, truncate 명령어는 삭제한 레코드를 복구할 수 없다.


# grant connect to hr
- hr 계정에게 데이터베이스를 접속할 수 있는 권한을 부여한다.


# revoke connect to hr
- hr 계정에게 데이터베이스를 접속할 수 있는 권한을 제거한다.


# grant resoure to hr
- hr 계정에게 자원을 조작할 수 있는 권한을 부여한다.


# revoke resource to hr
- hr 계정에게 자원을 조작할 수 있는 권한을 제거한다.


# to_char() 형식 문자
  - yyyy : 년도 4자리
  - mm : 월 2자리
  - dd : 일 2자리
  - hh : 시 2자리
  - hh24 : 시 2자리
  - mi : 분 2자리
  - ss : 초 2자리
  - q : 분기
  - day : 요일(풀네임)
  - dy : 요일(약어)
  - d : 요일(숫자)
